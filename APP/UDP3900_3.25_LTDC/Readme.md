# 🚀 ST7701S 切割屏高性能驱动实现 (STM32H750 + LTDC)

本项目记录了基于 **STM32H750** 驱动 $240 \times 960$ 特殊比例切割屏（原始逻辑分辨率为 $360 \times 960$）的底层实现与显示对齐优化方案。

---

## 🛠️ 1. 像素时钟 (DOTCLK) 与刷新率计算

根据 ST7701S 数据手册要求，像素时钟的最小周期时间 $T_{CYCD}$ 需满足 $\ge 33 \text{ ns}$。

* **时钟频率设定**：
  为了确保高分辨率下的画面平滑且无闪烁，我们将 LTDC 的像素频率设定在 $20 \text{ MHz}$ 至 $25 \text{ MHz}$ 之间。
* **参数验证**：
  当 $f_{CLK} = 25 \text{ MHz}$ 时，时钟周期 $T = 40 \text{ ns}$，完全符合手册中 $\ge 33 \text{ ns}$ 的时序规范。在此频率下，配合合理的 Blanking（消隐）参数，屏幕刷新率可稳定在 60Hz 左右。

---

## 📐 2. 物理映射与对齐策略 (Mapping Strategy)

该屏幕的物理显示区域为 $240 \times 960$，但驱动 IC 的逻辑宽度（Source Output）仍为 360 像素。这表明该屏幕在工艺上属于“物理切割屏”。



### 🔍 实验验证：背景对比法
1. **测试配置**：将 LTDC 背景色设为 **蓝色**，显存（SDRAM）填充色设为 **红色**。
2. **窗口映射**：通过调整 LTDC 图层的起始偏移量进行定位。
3. **关键代码配置**：
```c
/* 物理宽度 240, 逻辑宽度 360 -> 起始偏移量 = 360 - 240 = 120 */
pLayerCfg.WindowX0 = 150;  // 逻辑起始位置 (向右偏移)
pLayerCfg.WindowX1 = 359;  // 逻辑结束位置
pLayerCfg.WindowY0 = 0;
pLayerCfg.WindowY1 = 959;
```

### 💡 调试结论
实验观察到红色块完美填充屏幕，左侧逻辑区域显示为蓝色背景（物理不可见）。由此判定该屏幕为 **“左侧物理切割”**，显示内容需通过 LTDC 窗口配置进行 **右对齐** 处理。

## 🎨 3. 色彩格式与 SPI 初始化 (Pixel Format)

屏幕 IC 的色彩位宽需通过 SPI 初始化序列中的 `0x3A` 指令（Interface Pixel Format）进行定义：

| 指令 (Hex) | 参数 (Hex) | 色彩格式 | 备注 |
| :--- | :--- | :--- | :--- |
| **0x3A** | **0x55** | **RGB565 (16-bit)** | **本项目采用**：平衡刷新速度与显存开销。 |
| **0x3A** | **0x66** | **RGB666 (18-bit)** | 精度更高，但对数据总线负载较大。 |

## ⚠️ 避坑指南
1. **显存对齐**：由于 H750 具有 L1 Cache，在更新 SDRAM 显存后，务必调用 `SCB_CleanDCache_by_Addr()`，否则会出现花屏或画面残留。
2. **消隐参数**：特殊比例屏幕对 HSYNC/VSYNC 的宽度较敏感，建议优先参考屏幕出厂的参考代码（Init Code）进行微调。
