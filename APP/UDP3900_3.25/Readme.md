# 🚀 UDP3900 项目应用层 (APP) 核心框架模板

本项目是 **UDP3900 电源仪器** 应用层程序（Application）的起始原型。作为整机软件开发的基石，该模板集成了底层驱动初始化、系统安全防护及内存映射配置，为后续的业务迭代提供了稳定的运行环境。

---

## 🏗️ 1. 环境准备与引导机制

本程序采用 **“Bootloader + APP”** 的双段式架构，设计运行于外部 **QSPI Flash** 区域，不支持直接从内部 Flash `0x08000000` 地址启动。

*   **引导依赖**：在烧录本 APP 前，必须确保 MCU 已运行匹配的 Bootloader 程序。
*   **部署位置**：程序需通过指定的跳转协议或下载工具映射至 QSPI Flash 的特定偏移地址（基地址 `0x90000000`）。
*   **向量表重定向**：
    ```c
    /* 程序起始位置：QSPI Flash 偏移 0x2000 处 */
    SCB->VTOR = 0x90002000; 
    ```

## 🛠️ 3. 技术深度解析

### 3.1 缓存一致性与中断处理
为了保证从 Bootloader 跳转至 APP 后的环境稳定性，程序在入口阶段会强制刷新 Cache 并重新开启中断：

```c
SCB_DisableDCache();      // 关闭 D-Cache 以便进行清理
SCB_DisableICache();      // 关闭 I-Cache
SCB_InvalidateICache();   // 使指令缓存失效
SCB_InvalidateDCache();   // 使数据缓存失效
__enable_irq();           // 重新使能全局中断
```

### 3.2 固件元数据定义 (Firmware Info)
为了便于 Bootloader 进行版本校验与固件识别，元数据结构体被强制存放于 QSPI Flash 的固定偏移处。

*   **物理地址**：`0x90400000` (基地址 + 4MB 偏移)
*   **代码映射**：
    ```c
    /* 固件信息指针重定向至 QSPI Flash 的 4MB 标志位 */
    firmwareInfo = (FirmwareInfo_Def *)(0x90000000 + 0x400000ul);
    ```
